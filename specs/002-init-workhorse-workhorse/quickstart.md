# 快速入门: `init` 命令

本指南将引导您完成使用 `init` 命令初始化您的项目和全局配置。

## 先决条件
- `workhorse-cli` 已安装。

## 步骤

### 1. 首次初始化
在一个新的项目目录中，运行以下命令来创建全局和项目级配置：

```bash
workhorse-cli init
```

**预期输出**:
```
✔ 全局配置已在 ~/.workhorse 创建。
✔ 项目配置已在 ./.workhorse 创建。
初始化成功！
```

### 2. 处理已存在的配置
如果您再次运行 `init` 命令，系统将检测到现有配置并提示您进行确认：

```bash
workhorse-cli init
```

**预期输出**:
```
? 检测到现有配置。您想覆盖它吗? (y/N)
```
- 输入 `y` 将会重置配置。
- 输入 `n` (或按 Enter) 将会取消操作。

### 3. 强制覆盖
如果您希望在不收到提示的情况下自动覆盖现有配置，请使用 `--force` 标志：

```bash
workhorse-cli init --force
```

**预期输出**:
```
✔ 全局配置已在 ~/.workhorse 创建。
✔ 项目配置已在 ./.workhorse 创建。
初始化成功！
```

## 验证
成功执行后，您可以验证以下目录和文件是否已创建：
- **全局**: `~/.workhorse`
- **项目**: `./.workhorse/settings.json`

`settings.json` 文件最初将包含一个空对象:
```json
{}
```
